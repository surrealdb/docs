## `Duration`

When you receive a duration from SurrealDB, it will always be represented as a `Duration` class.

```python title="Signature"
Duration(duration: int)
```

### Create a `Duration`

```python
// Input in nanoseconds
dur = Duration(1000000);

// Parse surreal input, seconds and nanoseconds
dur = Duration.parse(1000, 200);

// By a unit amount
dur = Duration.nanoseconds(1000);
dur = Duration.microseconds(1000);
dur = Duration.milliseconds(1000);
dur = Duration.seconds(1000);
dur = Duration.minutes(1000);
dur = Duration.hours(1000);
dur = Duration.days(1000);
```

## `Geometry`

The Geometry types are `Point`, `Line`, `Polygon`, `MultiPoint`, `MultiLine`, `MultiPolygon` and `Collection`. The are all superclass of the `Geometry` class.


### Methods

Below, are all the methods implemented across all geometry derivatives.

#### `.get_coordinates()`

Used to convert a geometry to a Geo cordinates representation

```python title="Signature"
Geometry.get_coordinates()
```

```python title="Example"
line = GeometryLine(
    GeometryPoint(1, 2),
    GeometryPoint(3, 4),
)

line.get_coordinates()          // [ [1, 2], [3, 4] ]
```

### Derivatives

#### `GeometryPoint`

A point in space, made up of a long and lat coordinate, automatically converted to a float.

```python title="Signature"
GeometryPoint(long: float, lat: float)
```

#### `GeometryLine`

A line, made up of two or more points

```python title="Signature"
GeometryLine(GeometryPoint, GeometryPoint, ...GeometryPoint)
```

#### `GeometryPolygon`

A polygon, made up of self-closing lines

**Note**: The lines inside the polygon will automatically be closed if not already, meaning that the last point will be the same as the first.

```python title="Signature"
GeometryPolygon(GeometryLine, ...GeometryLine)
```

#### `GeometryMultiPoint`

A collection of one or more points

```python title="Signature"
GeometryMultiPoint(GeometryPoint, ...GeometryPoint)
```

#### `GeometryMultiLine`

A collection of one or more lines

```python title="Signature"
GeometryMultiLine(GeometryLine, ...GeometryLine)
```

#### `GeometryMultiPolygon`

A collection of one or more polygons

```python title="Signature"
GeometryMultiPolygon(GeometryPolygon, ...GeometryPolygon)
```

#### `GeometryCollection`

A collection of one or more `Geometry` derivatives

```python title="Signature"
GeometryCollection(Geometry, ...Geometry)
```

## `Table`

When you get a table name sent back from SurrealDB, it will be represented as a `Table` class.

```python title="Signature"
Table(table: str)
```

### Converting to string

```python
table = Table("table")
str(table)    # "table"
```

##  `RecordID`

When you receive a RecordID back from SurrealDB, it will always be represented as a `RecordID`. The class holds `tb` and `id` fields, representing the table name, and a unique identifier for the record on that table. A `RecordID` can be converted into a string, and will be represented as such when it's converted to JSON.

```python title="Signature"
RecordID(tb: str, id: Any)
```

### Working with a `RecordID`

```python title="Constructing"
# table is "person"
# unique identifier on the table is "john"
rid = RecordID("person", "john")
```

```python title="Extracting data"
# Simple
rid = RecordID("person", "john")
rid.tb # "person"
rid.id # "john"

# Complex
rid = RecordID("recording", { city: "London", data: 123 })
rid.id      # { city: "London", data: 123 }
rid.id.city # "London"
rid.id.data # 123
```

### Convert to String

```python title="Simple"
str(RecordID('table', 123))
# 'table:123'
str(RecordID('table', 'abc'))
# 'table:abc'
```

```python title="Complex characters"
str(RecordID('table', '123'))
# 'table:⟨123⟩'
str(RecordID('table', '123withletters'))
# 'table:123withletters'
str(RecordID('table', 'complex-string'))
# 'table:⟨complex-string⟩'
str(RecordID('table-name', 123))
# '⟨table-name⟩:123'
```

```python title="Objects and Arrays"
str(RecordID('table', { city: "London" }))
# 'table:{ city: "London" }'
str(RecordID('table', ["London"]))
# 'table:["London"]'
```
